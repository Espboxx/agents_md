# Lessons

- 这个仓库最可靠的基础验证不是运行端到端安装，而是先做 PowerShell Parser 语法检查；当前没有测试目录，Parser 是修改脚本后的最低门槛。
- 写入逻辑已经把“目录创建、UTF-8 无 BOM、内容比对、备份、覆盖/跳过”集中到 `Write-FileWithBackup`，后续扩展文件输出时应继续复用这条路径，避免绕开备份机制。
- `-Force` 只代表跳过确认并强制覆盖，不代表自动安装到当前项目；如果没有同时提供 `-RepoRoot`，项目级安装会被跳过，这一点在排障时必须优先确认。
- 当前目录已有 `AGENTS.md` 时，非 `-Force` 流程会额外询问“追加还是覆盖”；维护项目规范时应优先选择覆盖，避免重复模板被继续叠加。
